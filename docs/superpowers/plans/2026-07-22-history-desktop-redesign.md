# История на десктопе/планшете — редизайн (план)

Источник: консилиум (десктоп-синтез + планшет Вариант A) + мокапы
`wide-history-mockup.html`, `wide-history-tablet-mockup.html`,
`wide-history-finished-tournament.html`. Дизайн одобрен пользователем («A, делай»).

## Брейкпоинты (фиксированы в wide.js)
- <900 — телефон (без изменений).
- 900–1279 — планшет: **Вариант A** (сетка профилей отдельным экраном → лента участника → полноэкранная деталь).
- ≥1280 — десктоп: **колонка «Участники» │ лента │ деталь** (деталь = существующий TournamentView/GameCard, итоговая таблица уже там).

## Global Constraints
- Ничего из текущих функций Истории не терять: фильтр Все/Игры/Турниры; месяц ‹ › (фильтрует и сводку, и ленту); выбор участника (pFilter) → его матчи + В–П + ±дельты; лента игр(миксов)+турниров по датам с приглушением «не моих»; свайп-удалить/копировать; дрилл-ин в полный экран результата.
- Русский только через i18n (t()), ключи в ru/en/es. Без хардкода цветов темы. (ESLint-гейты.)
- Пуш только после зелёных eslint+build. Деплой — пользователь.

## Фаза 1 — Десктоп ≥1280: колонка «Участники» (ЭТОТ ПУШ)
Строгое супермножество: горизонтальную карусель выбора игрока на ≥1280 заменяем вертикальной колонкой (аватар + имя + В–П за месяц), выбранный подсвечен; логика pFilter та же; сводка месяца уже отражает выбранного (focusId). Лента и дрилл-ин — как есть (WideSplit/TournamentView).
- `wide.js`: `useIsWideXL()` (min-width:1280).
- `locales`: `hist_whose` («Чья история»).
- `HistoryView`: helper В–П по игроку за месяц; вертикальная колонка; карусель только при !xl; на xl layout = [колонка 236px | WideSplit(лента|деталь)].

## Фаза 2 — Планшет 900–1279: Вариант A
Сетка крупных карточек-профилей как состояние-лендинг → тап → лента участника (полоса статистики сверху) → тап события → полноэкранная деталь. Новые состояния навигации (grid ↔ memberFeed ↔ eventDetail), кнопки «назад». Видимые кнопки действий вместо свайпа на планшете.

## Фаза 3 — Десктоп: правый инспектор-статистика участника
Спарклайн рейтинга за месяц, В–П парой, форма-стрип, лучший партнёр/сложный соперник. Требует per-member дельты (доп. загрузка).
