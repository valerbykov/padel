# Публичные `/l/CODE` и `/j/CODE` — широкий экран — план

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** Десктоп-раскладка (≥900px) для публичной страницы лиги (`/l/`,
`LeaguePublicPage`) и приглашения в игру (`/j/`, `GuestJoin`); на `/j/` — явный выбор
напарника. Телефон (<900px) — прежняя вёрстка (кроме подписей выбора напарника на `/j/`).

**Архитектура:** переиспользуем `useIsWide()` из `src/components/wide/wide.js` и
двухколонный паттерн из уже сделанной `/t/`. Оба файла уже правились в Task 5 `/t/`-плана
(единый «Войти») — строим поверх.

**Tech Stack:** Vite + React 18 (JSX, без TS), i18n ru/en/es.

## Global Constraints

- Спек: `docs/superpowers/specs/2026-07-22-public-league-game-wide-design.md`.
  Макет: `scratchpad/guest-pages-wide-mockup.html` (одобрен).
- Единый «Войти» уже сделан (Task 5 `/t/`-плана: `onLogin` в `PublicToggles`, вход на
  `/l/` и `/j/`) — НЕ переделывать, только layout.
- `useIsWide()` вызывать в начале компонента, ДО ранних return (гейт хуков).
- Русский текст в JSX — только через i18n `t(...)` (ESLint `no-restricted-syntax`);
  латиница («VS») допустима. Цвета — только токены темы.
- Тест-фреймворка нет: проверка = `NODE_OPTIONS=--max-old-space-size=512 npm run build`
  без ошибок + `npx eslint src/` чисто + Playwright/скрин на 390px и 1280px (тёмная/светлая).
- Телефонную вёрстку не ломать. На `/j/` выбор напарника — осознанная UX-правка (и на телефоне).
- Натив не затронут (<900px). Деплой — вручную пользователем (вне задач).
