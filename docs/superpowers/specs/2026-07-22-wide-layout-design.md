# Широкоэкранная версия (iPad / планшеты / десктопный веб) — дизайн

Дата: 2026-07-22. Выбор пользователя после дизайн-совета (4 варианта):
**вариант B «морфящийся трёхколонник + инспектор» (Apple HIG) + ТВ-режим из варианта C**.
Макеты: артефакты `wide-variants-4` (сравнение) и `wide-layout-v1` (детализация вкладок).

## Цель

На ширине ≥900px приложение перестаёт быть «растянутым телефоном»: трёхколоночная
раскладка (рейл → список → деталь), премиум-слот «инспектор» справа, полноэкранное
ТВ-табло турнира для телевизора/планшета клуба. Телефонная версия (<900px)
**не меняется вообще**.

## Жёсткие ограничения (Global Constraints)

- `<900px` — существующая телефонная раскладка, ни одного изменения поведения.
- Деталь-компоненты (`GameCard`, `TournamentView`, `PlayerDetail`, `HistoryView`-деталь)
  переиспользуются **как есть**, внутри детали остаются колонкой `max-width: 520px`.
  Запрещено менять их вёрстку под ширину — всё широкое живёт в новых модулях-обёртках.
- Все новые части — **отдельные файлы/lazy-чанки** (`WideShell`, `TvBoard`,
  `InspectorPanel`): телефонная сборка не грузит широкоэкранный код.
- Никаких новых запросов данных для «пустых состояний» — только уже загруженные
  (bootstrap, swr-кэш лидерборда/списков). Урок совета: не строить поверхности впрок.
- Роутинг-регексы `/j/ /t/ /l/ /r/` в App.jsx не трогаем.
- Тёмная/светлая темы работают в обеих раскладках (переменные, как сейчас).

## Раскладка (≥900px)

Сетка: `[рейл 72px | список 320px | деталь 1fr (min 520px) | (инспектор 300px)]`.

- **Рейл (72px)**: иконки 4 вкладок, лого-лига сверху, аватар/профиль снизу.
  На ≥1280px морфит в сайдбар 244px (иконки + подписи + переключатель лиги);
  состояние запоминается в localStorage. Нижние табы при ≥900px скрыты.
- **Список (320px)**: текущие списки вкладок (игры, турниры, таблица лиги, лента
  истории) в существующем рендере. Выбранная строка — лаймовый индикатор слева.
- **Деталь (1fr)**: тот же компонент, что на телефоне. Второй уровень (из состава
  игры → профиль игрока) — стек ВНУТРИ колонки детали: кнопка «назад» в шапке
  детали откатывает стек, список не трогается. Реализация — поверх существующего
  `registerBack`/state, без URL-изменений в v1.
- **Инспектор (300px)**: выезжающая справа панель премиум-аналитики, привязанная
  к текущему выделению. **В v1 — за фичефлагом, по умолчанию ВЫКЛ** (совет обоих
  экспертов: пустую рамку не выпускаем). Шасси (панель, кнопка ✦, анимация)
  строится в v1, наполнение — отдельный этап «премиум-аналитика».

Пустое состояние детали (ничего не выбрано): лёгкая сводка из уже загруженных
данных — Игры: ближайшая игра + счёт live; Турниры: активный турнир; Друзья:
собственная карточка игрока; История: счётчики месяца из уже загруженной ленты.
Без новых RPC.

## Брейкпоинты

- `<900px` — телефон (без изменений).
- `900–1279px` — рейл 72px + список + деталь. Инспектор (когда включён) — overlay.
- `≥1280px` — рейл может разворачиваться в сайдбар 244px.
- `≥1440px` — инспектор (когда включён) докается четвёртой колонкой.
- iPad portrait (~768–834px) попадает в телефонную раскладку — осознанно (v1).

## ТВ-режим (из варианта C)

- Кнопка «📺 Табло» в афише турнира (виден организатору и участникам).
- Полноэкранный рендер: тёмный постерный фон, корты текущего раунда крупным кеглем,
  живая таблица, авторотация «корты ↔ таблица» каждые ~15с, QR-код ссылки
  `/t/CODE` в углу. Выход — Esc/крестик.
- **Публичный маршрут `/tv/CODE`** (новый регекс в App.jsx — добавление, не
  изменение существующих): переиспользует уже существующий публичный RPC
  `get_tournament_by_code` (как гостевая `/t/CODE`), обновление — поллинг 10–15с
  (реалтайм для анонима не гарантирован). Никакой новой SQL-работы.
- Модуль `TvBoard.jsx` — отдельный lazy-чанк, работает и как полноэкранный режим
  внутри приложения, и как публичная страница.

## Компоненты (новые файлы)

- `src/components/wide/WideShell.jsx` — грид, рейл/сайдбар, монтаж список+деталь.
  PadelLeague при `isWide` рендерит вкладки через WideShell; списки получают
  `onSelect`, деталь — существующие пропсы. Определение `isWide` — matchMedia
  `(min-width: 900px)` + resize-подписка.
- `src/components/wide/EmptyDetail.jsx` — пустые состояния из кэша.
- `src/components/wide/InspectorPanel.jsx` — шасси инспектора (за флагом).
- `src/components/TvBoard.jsx` — табло (in-app + `/tv/CODE`).
- Правки существующего: PadelLeague — развилка `isWide` в 4 точках рендера вкладок
  (деталь в панель вместо замещения); App.jsx — маршрут `/tv/`; nav скрыт при wide.

## Данные и состояние

- Выбор детали — существующие `selId/mode` и `sel` стейты; wide меняет только
  место рендера, не машину состояний.
- Смена лиги при открытой детали: деталь сбрасывается в пустое состояние
  (кэш per-league, деталь другой лиги невалидна).
- Диплинки (`/t/`, `/j/` ин-апп) на wide открывают соответствующую вкладку с
  выбранной деталью — тот же код, что сейчас (`openEvent`), плюс выделение строки.

## Ошибки/деградация

- При ошибке загрузки списка — существующие скелетоны/ошибки в колонке списка.
- ТВ-режим при потере сети показывает последние данные + бейдж «обновлено N мин
  назад» (данные и так в кэше поллинга).

## Этапность

- **v1 (этот план):** WideShell (рейл+список+деталь+пустые состояния) для 4 вкладок,
  морфинг рейла ≥1280px, ТВ-режим (кнопка + `/tv/CODE`), инспектор-шасси за флагом.
- **v2 (отдельный проект «премиум-аналитика»):** наполнение инспектора, докинг
  ≥1440px, URL-адресуемость выделения, iPad-таргет в Xcode + скриншоты сторов.
- Бэклог (не делаем без спроса): док закрепляемых виджетов и Cmd+K из варианта C;
  **freemium ТВ-режима** (напр. 30 мин бесплатно, дальше по подписке лиги) — гейтинг
  на сервере по entitlement лиги, строится вместе с механикой подписки в этапе
  «премиум-аналитика»; **нативное tvOS-приложение** (на tvOS нет WebView — Capacitor
  не применим; отдельный SwiftUI-проект на том же публичном RPC, делать при спросе
  клубов; сейчас табло на ТВ = AirPlay / браузер смарт-ТВ / Chromecast).

## Проверка

- `npm run build` + ESLint — чисто.
- Playwright: 390px (телефон не изменился — эталонные сценарии: списки, деталь,
