# Публичные `/l/CODE` и `/j/CODE` — широкий экран

**Дата:** 2026-07-22
**Статус:** дизайн согласован (макет `scratchpad/guest-pages-wide-mockup.html` одобрен). Реализация — ПОСЛЕ плана `/t/` (`2026-07-22-public-tournament-wide`). Отдельный план будет.

## Goal

Публичные гостевые страницы лиги (`/l/CODE`, `LeaguePublicPage`) и приглашения в
игру (`/j/CODE`, `GuestJoin`) получают десктоп-раскладку (≥900px). Телефон (<900px)
не меняется. `/r/` (ClaimProfile) НЕ трогаем — центрированная карточка корректна.
Единый «Войти» уже покрыт спеком `/t/` (cross-cutting раздел).

## `/l/CODE` — публичная страница лиги

Сейчас: одна колонка `max-width:440px`. Контент: шапка (лого клуба + название +
«N игроков · M игр · с даты» + канал Telegram), «УЧАСТНИКИ», подиум топ-3,
лидерборд с 4-го места, «Вступить в лигу» + OpenInApp + вирусный CTA «Создать лигу».

Десктоп (≥900px, `useIsWide`):
- **Шапка лиги** (лого + название + мета + Telegram) — на всю ширну сверху.
- Две колонки (грид `1fr 340px`, `align-items:start`):
  - **слева:** подиум топ-3 + лидерборд;
  - **справа (sticky):** «Вступить в лигу» + подсказка + OpenInApp + вирусный
    блок «Создать лигу».
- Телефон: как сейчас (стопкой).

## `/j/CODE` — приглашение в игру

Сейчас: одна колонка `max-width:400px`. Контент: «ТЕБЯ ЗОВУТ НА ПАДЕЛ» + лига,
title игры, дата/место, **корт 16:9 с 4 слотами** (A1/A2/B1/B2, `CourtSpot`),
подсказка «тап по свободному месту · N/4», имя гостя / индикатор входа, «Встать на
корт», ссылка «войти для рейтинга».

Десктоп (≥900px, `useIsWide`) — две колонки (грид `1fr 372px`):
- **слева:** eyebrow + title + дата/место + **корт** (визуальный герой);
- **справа (sticky):** запись.

**ВАЖНО — явный выбор напарника (правка UX, не только раскладка).** Сейчас кнопка
«Встать на корт» молча берёт `freeSlots[0]` — неясно, куда встанешь и с кем в паре.
Падел 2×2: сторона A = пара, сторона B = соперники. Делаем явным:
- на корте — бейдж **«VS»** по центру сетки; свободные места подписаны **«в пару к
  {имя}»** (партнёр = игрок на той же стороне);
- справа — выбор стороны кнопками: **«＋ В пару к {A} · против {B}»** (по одной на
  каждое свободное место/сторону), либо тап по месту на корте. Кнопка ведёт в
  конкретный слот (`take(team, position)`), а не в первый свободный.
- Телефон: как сейчас (корт + запись стопкой), но со свободными подписями «в пару
  к …» и явными кнопками выбора (не молчаливый `freeSlots[0]`).

i18n: новые ключи для «в пару к {name}» / «против {name}» / «VS» (ru/en/es).
Проверить, нет ли готовых (`pub_join_this_pair` = «Встать в пару к {name}» уже есть —
переиспользовать).

## Границы / не-цели

- `/r/` ClaimProfile — без изменений (центрированная карточка).
- Единый «Войти» — в спеке `/t/` (добавляется в `PublicToggles`; здесь только
  прокидываем `onLogin`).
- Натив не затронут (та же вёрстка <900px).
- Реализация — отдельным планом ПОСЛЕ завершения плана `/t/`, переиспользуя тот же
  `useIsWide` / двухколонный паттерн.

## Риски

- `/j/`: смена поведения кнопки записи (первый-свободный → явный слот) — проверить,
  что `take(team, position)` работает для выбранного слота и на телефоне тоже.
- ESLint-гейты: русский только через i18n; токены темы.
