# Тумблер «маскот вкл/выкл» на лиге — первый срез движка тем

**Дата:** 2026-07-22
**Статус:** согласован (тумблер бесплатный; нейтральная лестница одобрена).
Первый шаг [[white-label-theming]]. Реализация — ПОСЛЕ Task 4 витрины (конфликт локалей).

## Goal

На «Управлении лигой» — простой переключатель **«Маскот (собаки)»** вкл/выкл
(дефолт ВКЛ). При ВЫКЛ активная лига теряет собачий мотив: аватары-собаки → нейтральные
инициалы-кружки, а собачий словарь → нейтральный. Бесплатно для всех. Применяется по
**активной** лиге (как вся будущая темизация).

## Модель

- Поле `mascot boolean not null default true` на `groups` (миграция).
- App.jsx при выборе активной лиги выставляет рантайм-флаг темы (как будет для всей
  темизации). Аватары и словарь читают этот флаг.
- Гейтинг: НЕ Pro — доступно всем. (Цвета/логотип/поддомен — отдельно, Pro.)

## Что переключается

### 1. Аватары (`src/lib/avatar.js`)
Сейчас `dogAvatar/playerAvatar/avatarFallback/avatarBg` возвращают собак. Добавить
режим «без маскота»: вместо собаки — **инициалы на детерминированном по имени фоне**
(SVG data-URI или CSS-фон + буквы; цвет — хэш имени). Реализация: модульный флаг
`setMascotEnabled(bool)` (App выставляет при смене активной лиги), функции ветвятся.
`playerAvatar(url, idOrName)` при пустом url и mascot=off → инициалы. Реальное
загруженное фото игрока показывается всегда, независимо от флага.

**Тумблер управляет только АВТОЗАГЛУШКОЙ; личный выбор всегда побеждает
(решение 2026-07-22, вариант B).** Модель как с фото: явно выбранный аватар
показывается всегда, тема лиги задаёт лишь дефолт для тех, кто НИЧЕГО не выбрал.
- **Выбор собак-пресетов в профиле ОСТАЁТСЯ** (`ProfileEditor.jsx`, `PRESETS`) —
  выбранная собака хранится как `avatar_url` (`/avatars/dog-NN.webp`) и показывается
  во всех лигах, включая no-mascot. Как и загруженное фото.
- `mascot=off` меняет ТОЛЬКО поведение при **пустом** `avatar_url`: `playerAvatar(url,
  idOrName)` при `!url` → инициалы вместо `dogAvatar`. То же для `avatarBg`/`avatarFallback`.
- Никакой миграции данных и «схлопывания» дог-URL не нужно — существующие собачники
  сохраняют свою собаку.
- **Честный минус:** no-mascot лига не на 100% без собак — у тех, кто сам выбрал
  собаку, она остаётся среди инициалов. Это их явный выбор (как фото); принято.

### 2. Словарь (i18n, ru/en/es)
Нейтральные варианты (mascot=off) для:
- **Тир-лестница** (6): Щенок→**Новичок**, Юниор стаи→**Любитель**, Игрок стаи→**Игрок**,
  Охотник→**Опытный**, Хищник→**Мастер**, Вожак стаи→**Чемпион** (+ short-формы).
- **Слоган** `tagline`: «лига друзей · своя стая на корте» → «лига друзей · своя игра на корте».
- **Футер** `sc_footer`: «лига твоей стаи 🐕» → «твоя лига 🎾» (или без эмодзи).
- Прочие вкрапления 🐕/«стая» в JSX (демо-герой `demo_hero_*`, `feat_demo_title`,
  `w_pod_hint`) — нейтральные варианты.

Реализация словаря: ключи-варианты (`tier_puppy` / `tier_puppy_neutral` …) и хелпер
`tierName(key, mascot)` ИЛИ отдельная секция ключей, выбираемая по флагу. Выбрать
минимально-инвазивный путь; не плодить дубли там, где текст и так нейтрален.

### 3. Эмодзи-иконки
🐶 в тир-бейджах / 🐕 в акцентах при mascot=off → нейтральные (🎾 или без эмодзи).

путём сохранения настроек лиги (права — как у прочих настроек: владелец/организатор).

## Границы v1

- **Скоуп — ин-апп (активная лига).** Публичные страницы (`/l/`,`/t/`,`/j/`) в v1
  остаются на домашнем маскоте (их RPC пока не отдают `mascot`). Прокинуть флаг в
  публичные RPC — fast-follow (одна строка на RPC + чтение на клиенте); отметить в плане.
- Цвета/палитра/логотип/поддомен — НЕ здесь (Pro, отдельные спеки).
- Полноценный «человеческий» набор картинок-аватаров — бэклог; здесь инициалы.

## Риски

- `avatar.js` импортируется широко — модульный флаг должен выставляться ДО первого
  рендера ростера при смене лиги (App выставляет синхронно из bootstrap/выбора лиги),
  иначе мигание. Проверить порядок.
- ESLint-гейты: русский только через i18n; токены темы.

## Тестирование

- Build + eslint. Playwright: лига с mascot=on (собаки, «Вожак стаи») и mascot=off
  (инициалы, «Чемпион») — вкладки Друзья/История/Турниры; переключение в «Управлении»
  перекрашивает без перезагрузки. Тёмная/светлая.
