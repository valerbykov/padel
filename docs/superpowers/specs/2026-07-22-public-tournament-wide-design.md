# Публичная гостевая страница турнира `/t/CODE` — афиша, профили, широкий экран

**Дата:** 2026-07-22
**Статус:** согласован дизайн (макет v4 одобрен), готов к плану.

## Goal

Привести публичную гостевую страницу турнира (`/t/CODE`, незалогиненный) к
согласованному дизайну приложения и раскрыть её на десктопе: **афиша + участники**
(строками на телефоне, большими профилями-карточками на десктопе), бейджи уровня
игроков, двухколонная раскладка ≥900px, одно «Войти».

## Контекст и проблема

`/t/CODE` рендерит `src/components/TournamentJoin.jsx`. Для **открытого** турнира
(`status==="open"`, гость не записан) он показывает **свою старую чиповую
join-карточку** и намеренно НЕ рендерит `TournamentView` (строки 349–352: «join-карточка
уже содержит всю инфу, не дублируем афишей ниже»). Из-за этого:

1. **Нет согласованной афиши** — постер `.trp-poster` (единый header турнира) на
   гостевой не показывается для открытого турнира.
2. **Нет десктоп-раскладки** — `.tj-wrap{max-width:460px}` — телефонная колонка,
   на десктопе пусто по бокам.
3. **Дублирование «Войти»** — кнопка в углу (`PublicToggles`) + ссылка «Войти для
   записи в рейтинг» у формы.

Для остальных статусов (active/finished/joined) гостю уже отдаётся
`<TournamentView readOnly spectatorMode initialT={t} />` — там афиша есть. Расходится
только открытый/набор.

Дизайн-мотив собачьей «стаи», аватары-собаки (`lib/avatar.js`), CSS-токены темы —
всё как в приложении, не меняем.

## Согласованный дизайн (макет v4)

Ссылка на макет: `scratchpad/t-guest-wide-mockup.html` (v4).

- **Телефон (<900px):** афиша `.trp-poster` → карточка «Участники» **строками**
  (аватар + имя + бейдж уровня в конце строки) → дашед-строки свободных мест →
  поле «Твоё имя (как гость)» + «Присоединиться к турниру» + подсказка.
- **Десктоп/планшет (≥900px):** две колонки —
  - **слева:** компактная афиша + сетка **больших профилей** (аватар 58px + имя +
    бейджи Playtomic/Lunda) + дашед-карточки свободных мест;
  - **справа:** карточка записи (прогресс-бар, имя гостя, кнопка, подсказка), sticky.
- **Бейджи уровня** — Playtomic (синий) / Lunda (салатовый) по реальным иконкам
  (`LevelBadges`). Нет уровня → без бейджа.
- **Одно «Войти»** — только в верхнем углу (`PublicToggles`). У формы вместо второй
  кнопки — тихая подсказка: «Как гость — без учёта в рейтинге. **Войти** сверху,
  чтобы результат засчитался.»
- **Свободные места** — дашед-строки (телефон) / дашед-карточки (десктоп).
- **Тема** — тёмная и светлая наследуют существующие токены.

## Архитектура

### Данные — публичный RPC

`migrations/…get_tournament_by_code…` — в агрегат `players` добавить поле
`'levels', p.levels` (рядом с существующим `'avatar_url', p.avatar_url`; left join
`profiles p` уже есть). `profiles.levels` — `jsonb` `[{"sys":"pt|ltr|oth","val":…}]`,
дефолт `[]`. Уровни помечены «видно всем» (`pc_level_hint`) — приватности нет.
**Только это поле; рейтинг/звания НЕ добавляем** (лига-скоуп, чувствительнее, у
гостей пусто). Функция остаётся `security definer`, grant `anon, authenticated` — без
изменения прав. Деплой миграции — вручную (прод-БД, Mac пользователя).

### Компоненты

- **Афиша — один источник правды.** Вынести блок `const afisha` из `TournamentView`
  (`Tournaments.jsx`) в экспортируемый компонент `TournamentPoster` (пропсы: `trn`,
  `readOnly`, экшены share/tv/delete опционально). `TournamentView` рендерит его же —
  визуально идентично (проверить парити в приложении). Публичная страница
  переиспользует его.
- **Публичный ростер участников** — новый презентационный компонент
  `PublicRoster({ players, targetSize, isWide, isPair })`:
  - `isWide` → сетка карточек (аватар + имя + `LevelBadges` + дашед-карточки на
    `targetSize − players.length` свободных мест);
  - `!isWide` → строки (аватар + имя + `LevelBadges` в конце) + дашед-строки.
  - read-only: без ✕/AddPlayer. Пары (`fmt.category==="pair"`) — по парам, как в
    приложении.
- **Раскладка** — в `TournamentJoin` для открытого гостя использовать `useIsWide()`:
  `isWide` → `WideSplit`-подобный грид `1fr / 372px` (слева афиша+`PublicRoster`,
  справа join-карточка sticky); `!isWide` → стопкой (афиша, `PublicRoster`,
  join-блок). Логику записи (`join()`, `name`, гость/сессия) не меняем — только
  презентацию.
- **Убрать** старую чиповую join-карточку для открытого турнира; для открытого
  статуса всегда показывать афишу + `PublicRoster` + join-блок.
- **Одно «Войти»** — удалить `tj-loginlink` «Войти для записи в рейтинг», добавить
  подсказку `jhint`. Ключ i18n: заменить/добавить `pub_guest_rating_hint`
  (ru/en/es).

### Что НЕ трогаем (границы)

- Активный/завершённый публичный вид турнира (матчи/сетка/итоги) — остаётся как
  сейчас (телефонная ширина). Широкая раскладка для них — отдельный бэклог.
- Публичная страница лиги `/l/CODE` — аналогичная адаптация отдельным спеком (бэклог).
- Рейтинг/звания на публичной — вне скоупа.
- Нативные приложения функционально не затронуты (та же телефонная вёрстка <900px).

## Риски

- **Прод-миграция RPC** — деплоит пользователь; сверить, что `levels` приходит и
  старые клиенты не ломаются (лишнее поле игнорируется).
- **Вынос афиши из `Tournaments.jsx`** (2400 строк) — риск регресса in-app афиши;
  обязательна проверка визуального парити в приложении (все статусы).
- ESLint-гейты репо: русский текст только через i18n-ключи; хардкод цветов темы
  запрещён — использовать токены.
