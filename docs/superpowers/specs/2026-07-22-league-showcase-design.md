# Витрина лиги: «Афиша-герой + живая лента» на `/l/CODE` — фаза A

**Дата:** 2026-07-22
**Статус:** дизайн одобрен (консилиум → макет `scratchpad/league-hero-feed-mockup.html`;
скины подтверждены `storefront-skins-mockup.html`). Фаза A из программы
«афиша + модерация + очередь» (см. спек `2026-07-22-public-tournament-wide-design.md`
и решения по видимости/заявкам/очереди).

## Goal

Публичная страница лиги `/l/CODE` становится витриной организатора: ближайшее
событие — постером-героем, остальные — живой лентой с записью в один тап.
**Фаза A — только показ и переходы** (запись происходит на существующих `/t/` и
`/j/`); модерация заявок — фаза B, очередь — фаза C.

## Ключевые решения (утверждены ранее)

- Афиша показывает **турниры И игры** (открытые/идущие).
- **Тумблер «Показывать в афише лиги»** per-событие, дефолт ВКЛ — делаем сразу в A.
- Полное событие в фазе A — ярлык «полный» и **без кнопки** (кнопка «В очередь» — фаза C).
- **«Пульс лиги»** — только из реальных записей (никакой симуляции).
- Витрина **free для всех** в домашней теме; брендинг — отдельная Pro-фича
  ([[white-label-theming]] в памяти). Строим на CSS-токенах.
- Двухколоночная оболочка `/l/` (Task 1 плана lj-wide) сохраняется.

## Дизайн (по одобренному макету)

**Десктоп (≥900px), левая колонка:**
1. **Афиша-герой** — ближайшее открытое событие постером: бейдж «Сбор открыт»
   (пульсирующая точка), Anton-титул, подзаголовок, мета-строки (дата/место/
   уровень/взнос), стек аватаров идущих + «N идут», строка пульса
   («⚡ {имя} записался {n} мин назад»), крупный счётчик «осталось N мест»,
   CTA «Записаться» → `/t/CODE` или `/j/CODE`. Диагональная полоса, маскот.
2. **«Дальше в лиге»** — таймлайн остальных: дата-якорь (число+месяц, Anton),
   название, мета (время·место·уровень·взнос), прогресс-бар заполнения
   (красный при <3 мест), стек аватаров, кнопка «Записаться»/«N мест»;
   полный → ярлык «полный», без кнопки; идёт → ярлык «идёт», кнопка «Смотреть».
3. **«Было круто»** — корешки последних завершённых (ч/б-стиль, эмодзи+название+дата).
4. Подиум топ-3 + свёрнутый «Полный рейтинг · N игроков» (existing лидерборд
   под разворотом) + **«Вся стая»** — стена аватаров участников (+N).

**Правая sticky колонка:** карточка клуба (лого, название, метрики, Telegram),
CTA «Записаться на {ближайшее}» + «Вступить в лигу», 1–2 строки пульса.

**Телефон (<900px):** герой → таймлайн → «Было круто» → подиум/рейтинг →
стена → существующий join-блок; **липкая нижняя CTA** «Записаться на {событие} ·
осталось N мест».

**Cold start:** нет открытых событий → героя нет; показываем маскота +
«скоро первое событие» + Telegram; остальная страница как сейчас.

## Данные

### Миграция 1: тумблер афиши
`tournaments` и `games`: `add column if not exists listed boolean not null default true`.

### Миграция 2: расширение `get_public_league`
Текущий RPC (см. `migrations/2026-07-12_public_tournament_avatars.sql`,
SECURITY DEFINER, grant anon) отдаёт группу+members+counts. Добавить:

- `'events'`: массив ближайших событий (turnaments `status in ('open','active')`
  + games `status = 'open'`), `listed = true`, отсортированных по `starts_at`
  (NULL — в конец), лимит ~10. Поля на событие: `kind` ('tournament'|'game'),
  `id`, `invite_code`, `name/title`, `starts_at`, `place`, `status`,
  `format` (у турниров), `level` (jsonb события), `fee_per_player`+`fee_currency`,
  `taken` (занято), `target` (мест всего; у игр = 4), `going` (до 5 аватаров:
  name+avatar_url), `last_join` (имя + created_at последней записи — для пульса).
- `'recent'`: 3 последних завершённых события (kind, название, дата, эмодзи-формат,
  победитель по имени — если дёшево; иначе без победителя).
- Существующие поля не менять (обратная совместимость: старые клиенты игнорируют).

Экспозиция данных: только то, что уже публично на `/t/`/`/j/` (имена, аватары,
взнос, места). Ничего нового чувствительного. Пульс — имя+время записи (имя и так
в ростере события).

### Клиент
`getPublicLeague` в `padelApi.js` — тип возврата расширяется прозрачно.

## UI-изменения

- `src/components/LeaguePublicPage.jsx` — секции афиши по макету (герой,
  таймлайн, корешки, стена, sticky-CTA на телефоне). Свёртка лидерборда.
  Переиспользуем токены; кнопки ведут на `/t/{code}` / `/j/{code}`.
- **Тумблер** «Показывать в афише лиги» в создании/редактировании турнира
  (`Tournaments.jsx` wizard) и игры (создание игры в `PadelLeague.jsx`) —
  чекбокс, дефолт вкл; апдейт поля `listed`.
- Относительное время пульса («4 мин назад») — хелпер с i18n (ru/en/es),
  без библиотек.

## Не-цели фазы A

- Запись/очередь/заявки прямо с витрины (переходим на `/t/`//`/j/`).
- Realtime-обновление пульса (данные на момент загрузки; поллинг не добавляем).
- Брендинг/темы (Pro, отдельно), фильтры событий, `/duck`-слаги.

## Тестирование

- Build + eslint (гейты: i18n-ключи, токены).
- Playwright: `/l/CODE` реальной лиги — 390px (стопка+липкая CTA) и 1280px

  полное событие (ярлык, без кнопки), тёмная/светлая.
- Прод-миграции применяю через Supabase MCP с проверкой (как levels).
