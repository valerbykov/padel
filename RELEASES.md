# Релизы PadelPack

Трекер новых фич и того, что уже выкачено в нативные приложения. Ведётся в
репозитории (не в памяти) — виден всем и переживает сессии.

**Что такое релиз:** это когда собрано несколько пушей, поднят номер build-версии
натива (`android/app/build.gradle` versionCode / iOS build number) и приложения
обновлены в сторах. **Push в git ≠ релиз** — пуши просто копятся.

**Как ведём:**
- Каждую новую фичу дописываем в **«Ожидает релиза»** (коммит + что нужно
  задеплоить на сервере: миграции/edge-функции).
- Слово **«релиз»** от пользователя = сигнал: поднял версию и собрал натив →
  переносим накопленное из «Ожидает релиза» в **«История релизов»** с версией и датой.
- Серверные части (миграции, edge-функции) деплоятся вручную и могут не совпадать
  по времени с натив-релизом — отмечаем отдельно.
- Работаем только в `main` (ветку `edge-to-edge` закрыли).

**Каналы Android:** Google Play и **RuStore** — один и тот же AAB/APK-код, но
подписи разные (RuStore переподписывает своим ключом; его SHA-256 добавлен в
assetlinks.json и должен быть в Яндекс OAuth-консоли). Релизы по каналам могут
идти не синхронно — в истории помечаем, куда выкачено.

## Ожидает релиза

- **SMS-вход убран совсем** — вкладка/поле телефона удалены ещё ранее; остался
  вход по паролю (ревью Apple + админ-аккаунты). 2026-07-20 дочищены последние
  мёртвые комменты. Веб уже без SMS ✓; **на телефоне SMS исчезнет после пересборки
  натива** (старый бандл ещё показывает). Edge `send-sms-hook` оставлен (безвреден,
  UI его не зовёт) — при желании отключить провайдер phone в Supabase отдельно.
- **Perf гостевого лендинга 66→97** — `9fa2773`, `7f5e53a`: экран «Начало»
  (`WelcomeScreen`) вынесен в отдельный чанк (~6K) — гость больше не грузит весь
  PadelLeague+Tournaments; LCP-аватары подиума не lazy + 104px-варианты. Веб
  задеплоен ✓ (Netlify Lighthouse Perf 97); натив — пересборка.
- **RuStore-канал: подготовка** — `79d693d`: SHA-256 подписи RuStore добавлен в
  assetlinks.json (App Links на RuStore-сборках) — уже на проде ✓. Отпечаток RuStore
  (`43:7C:…`) в Яндекс OAuth-консоли уже есть (внесён 2026-07-16) ✓; при интеграции
  Yandex SDK свериться, что там все три (CA:5B, 43:7C, E2:EF) — см. YANDEX_NATIVE.md.
- **Взносы за турнир** — `9e7f9a6` + `11bc4dd`. Карточка в завершённом турнире:
  сумма (общая или с каждого), участник сам отмечает «скинулся», админ проверяет,
  прогресс; **«Напомнить пушем»** — адресное уведомление на локскрин каждому
  должнику (троттлинг час), «в чат» — запасная кнопка (с пояснениями под обеими);
  **сбор запускается явно** кнопкой организатора; **пуш всем при завершении**
  игры/турнира («хорошего дня/вечера» по локальному часу + намёк про взнос,
  триггеры БД); **взносы и для игр** (общая FeesCard в сыгранной игре/миксе);
  **авто-сообщение в Telegram-чат лиги** при пуш-напоминании (публичный
  t.me/username + бот в чате). Сервер: миграции взносов `2026-07-24/25/27` все
  применены ✓ и **редеплой `send-due-reminders`** сделан ✓ (сверено в прод-БД
  2026-07-20: колонки/таблицы/RPC на месте). Веб задеплоен ✓; натив — пересборка.

- **Гость: ссылка + QR для регистрации игрока** — `0777edd`…`6ca669a`. Фикс шаринга
  на нативе (Capacitor Share) + QR claim-ссылки; кнопка «Открыть в приложении» на
  /r и /l (padelpack:// по тапу) с таймер-фолбэком «Скачать из стора»; починены
  ensure_claim_code (uuid) и claim_profile (через merge_profiles) — миграции
  применены ✓. Веб-части — деплой Netlify; шаринг на нативе — пересборка натива.
- **Пуши: устойчивость регистрации токена** — `ff460f4` (ядро токена сохраняется,
  даже если колонка tz/lang ещё не в БД). Пересборка натива.

- **Локализация пуш-уведомлений (ru/en/es)** + обновление языка при смене языка в
  приложении — `16b9483`.
  Сервер: миграция `2026-07-22_push_token_lang.sql` + редеплой `send-due-reminders`.
- **Yandex-вход: системный браузер закрывается после успешного входа** — `78dc631`.
  Только пересборка натива.
- **Нативный Yandex-вход (Yandex ID SDK, выбор аккаунта)** — `212d0e4` (JS+edge).
  Нативная часть — по `YANDEX_NATIVE.md`: SDK Android/iOS, регистрация в Yandex OAuth,
  редеплой `yandex-auth`. Без неё работает веб-фолбэк.

## История релизов

### iOS 1.0 — App Store (~2026-07-16) — первый публичный релиз 🎉
Одобрено Apple. Состав — весь функционал до недавних коммитов (см. `FEATURES.md`).
Свежие пункты из «Ожидает релиза» (локализация уведомлений, нативный Yandex-вход,
облегчённый TG-мост) в эту сборку НЕ вошли — идут в следующую.

Android — публикуется отдельно (versionCode растёт в `android/app/build.gradle`).
Более ранний функционал (до трекера) — в `FEATURES.md`, без привязки к версиям.
